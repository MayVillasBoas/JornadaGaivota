# Design: Decision Copilot UX Redesign — Flowing Session with Organic Visuals

Generated by /brainstorming on 2026-03-21
Branch: feature/copilot-ux
Status: APPROVED

## Problem Statement

The Decision Copilot MVP works functionally but the UX fails at engagement. Real user testing revealed 6 critical issues:

1. **Too long** — 18 questions across 4 modules is exhausting
2. **No voice input** — voice-input.ts and VoiceInput.astro exist on the site but aren't integrated
3. **Static experience** — textarea → click next → repeat feels like a form, not a guided session
4. **No AI feedback** — insights only appear on the NEXT step, easy to miss. No streaming.
5. **Disconnected from site** — doesn't match the organic, warm aesthetic of the rest of May.labs
6. **Nobody will finish** — no engagement hooks, no visual reward, no sense of progress

## Solution: "Flowing Session"

Transform the copilot from a multi-page form into a flowing, single-scroll conversation with organic visual elements that grow as the person progresses. Voice-first input. Streaming AI reflections after every response. Questions cut in half.

## Design Decisions

### 1. Organic Visual System — "Breathing Line"

A generative organic line runs along the left edge of the session timeline, inspired by the mycelium visualization in the Lab. It grows with each response:

- **Growth:** The line extends downward as the user progresses, with organic branching at key moments (theme nodes, module transitions)
- **Color by layer:** Warm coral (#c27a5a) for FEEL → Lilás (#8b6aad) for SEE → Blue-green (#4a7aad) for THINK → Deep green (#2B4A3E) for ACT
- **Theme nodes:** When the AI extracts themes, they appear as small nodes on the breathing line with keywords floating beside them. Repeated themes across modules connect visually.
- **Module transitions:** The line curves/spirals and shifts color with a ~1s animation
- **Implementation:** Canvas element positioned absolute on the left side, ~40px wide. Uses 2D canvas with bezier curves and noise-based organic movement. Lightweight — no WebGL needed.

### 2. Interaction Pattern — One Question at a Time

Each interaction follows this flow:

```
Copilot asks question (serif, large, appears with fadeIn)
    ↓
User responds (voice or text, sticky input at bottom)
    ↓
AI reflection streams in immediately (letter by letter, accent card with ✦)
    ↓
Theme nodes appear on breathing line
    ↓
Next question fades in (after ~1s pause)
```

No "next" button between questions within a module. The flow is automatic after the AI reflection completes. Between modules, a brief organic transition with the option to pause.

### 3. Voice-First Input

- **Mic button** is the primary action — large, prominent, left of textarea in sticky bottom bar
- **Waveform animation** during recording (using existing voice-input.ts patterns)
- **Transcription indicator** — "voice · 32s" shown under the user's response
- **Textarea** as secondary input — smaller, placeholder "type or tap the mic..."
- **Integration:** Reuse existing `voice-input.ts` module and `VoiceInput.astro` component patterns

### 4. Streaming AI Reflections

After each user response, the AI reflection appears with streaming animation:

- Call `/api/copilot` with action `reflect`
- Display response character by character (~30ms per character) in the accent card
- Card fades in first (empty), then text streams in
- Theme tags appear after streaming completes
- This creates a feeling of the AI "thinking with you" rather than returning a static block

### 5. Reduced Question Count

Each module is cut to 2 core questions. The AI compensates with deeper, more provocative reflections.

**Body Scan (FEEL) — 2 questions:**
1. "Close your eyes. Imagine choosing the first path. What happens in your body?"
2. "Now imagine the other path. What does your body do? Which felt more like expansion?"

**Parts Mapping (SEE) — 2 questions:**
1. "There's a voice that wants one thing, and another pulling differently. Describe them both — what does each want? What does each fear?"
2. "Stepping back from both parts — what do you notice? Is there a way to honor what both actually need?"

**First Principles (THINK) — 2 questions:**
1. "Write down every belief about this situation — every 'I have to,' 'I can't,' 'they expect me to.' Then mark: which are facts and which are assumptions?"
2. "Looking at just the facts — what's one assumption you've been treating as fact that would change everything?"

**Regret Minimization (THINK) — 2 questions:**
1. "You're 80, looking back. You chose path A and lived with it for decades. Then imagine you chose path B. Which regret feels heavier?"
2. "What would 80-year-old you tell present-day you?"

**Decision Memo (ACT) — 2 questions:**
1. "Based on everything — body signals, inner parts, facts vs assumptions, regret — what are your REAL options and their trade-offs?"
2. "What's the smallest next step you can take in the next 24 hours?"

**Total: 10 questions** (down from 18-21)

### 8. Mobile Considerations

- Breathing line becomes thinner (1px) and hugs the left edge
- Input bar stays sticky at bottom
- Voice button is full-width tap target when recording
- Theme nodes stack vertically instead of floating

### 9. Tracking Events (unchanged)

Same events as MVP: session_started, intake_completed, classification_shown, module_started, module_completed, session_returned, memo_generated, memo_exported.

## What's NOT Changing

- Classification engine and routing logic (working fine)
- Persistence layer (localStorage)
- Tracking events
- API endpoint structure
- Module framework (Body Scan, Parts Mapping, First Principles, Regret Minimization, Decision Memo)

## Success Criteria

- [ ] Session feels like a flowing conversation, not a form
- [ ] Voice input works as primary input method
- [ ] AI reflections stream in after every response
- [ ] Breathing line grows organically through the session
- [ ] Total questions ≤ 10 (down from 18-21)
- [ ] Visual aesthetic matches rest of May.labs (warm, organic, serif)
- [ ] Mobile experience is smooth
- [ ] User can complete entire flow in one sitting (~15-20 min)
